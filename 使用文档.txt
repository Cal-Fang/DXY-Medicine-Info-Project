以下默认python3

STEP1 
python getIP.py
可以多运行几次，每次添加100条待测IP

STEP2
python testIP.py
将能用的IP写入goodIP.txt

STEP3
python medicalSpider.py
其中主要两个步骤，建议一次跑一个函数，先注释掉另一个：
第一步step0()，得到temp.txt，为链接+药物名+公司+商品名，这一步应该不会被封
第二步step1()，得到temp2.txt，为药物名+商品名+成份+适应症，这一步因为爬取具体药物内容，即使动态IP+随机UserAgent+放慢爬取频率还是有被封的危险（不知道采用的是什么反爬技术，自动填充验证码太难实现），被封时报错如下：
Traceback (most recent call last):
  File "medicalSpider.py", line 185, in <module>
    step1() #得到temp2.txt,为药物名+商品名+成份+适应症
  File "medicalSpider.py", line 171, in step1
    raw_data = parseHtml2(getHtml(u, url))
  File "medicalSpider.py", line 59, in parseHtml2
    return raw_result[0]
IndexError: list index out of range
这是由于返回了验证码页面的缘故。验证码页面存于error.html，此时只需浏览器打开文件夹中的error.html进入页面手动填写验证码，并且最好更换一批IP（重复STEP1&2）再继续爬取即可。程序会在中断页面继续爬取，不会重复信息。

STEP4
python dataHandle.py
进行数据处理：
第一步对temp.txt和temp2.txt去重，因为有些药物适应多个categories，得到data0.txt和data1.txt
第二步整合两个txt中的信息，若药物名与商品名均相同则应视为同一种药物，得到data2.txt，内容为药物名+商品名+成份+适应症+公司+链接，其中公司和链接可以有若干个，代表不同商家生产的同一种药物
第三步剔除无效记录+再去重，得到data3.txt
第四步由data3.txt生成最终data.json标准格式文件，其格式示例如下：
{"name":"卡维地洛片", "tradeName":"妥尔", "ingredient":"本品主要成份为卡维地洛。", "use":"轻-中度原发性高血压。", "factories":["海南碧凯药业有限公司"], "url":["http://drugs.dxy.cn/drug/115489.htm"]}
注意：由于少部分页面格式不符合规律，爬取记录中存在一些不规则记录，程序运行会在这些记录上遇到错误而终止，这时需要手动调整该记录格式或删除再运行。全部错误记录修改完毕后得到的文件命名为data4.txt已附在文件夹中，以供参考。
